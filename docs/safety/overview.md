# Safety Overview

Reference: [Safety Index](./index.md)

## Purpose

This directory stores the approved final-state safety and security view for the project.

## Current Scope

The safety documentation maintained here may cover:

- threat modeling
- mitigation strategies
- privacy and sensitive-data risks
- security hardening expectations
- AI safety and guardrail concerns
- protective interaction and system controls
- MVP-specific trust boundaries and safe-handling rules

## Documentation Principle

Only approved final safety documentation belongs here.
Working notes, rejected variants, and temporary exploration are intentionally excluded.

## Maintenance Rule

The safety agent maintains this documentation in parallel with approved safety-related changes.
